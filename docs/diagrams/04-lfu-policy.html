LFU 策略数据结构与原理

双 HashMap + 频次桶 + minFrequency · 最少使用优先淘汰 · 同频次内保持先后顺序

STRUCTURE
数据结构映射
LFU 在本项目中没有复用链表基类，而是单独维护两层索引： frequencies 记录每个 key 当前访问次数； keysByFrequency 按频次把 key 放入桶中； minFrequency 始终指向当前最小频次。 这样访问时可以从旧频次桶移到新频次桶，淘汰时直接从最小频次桶中拿第一个 key。
frequencies: Map keysByFrequency: Map> minFrequency = 最小桶号 evictKey() = min 桶首元素
代码语义
1. put 时把新 key 放入频次 1 的桶。
2. access 时频次 +1，并从旧桶迁移到新桶。
3. 如果旧桶空了，要更新 minFrequency。
4. 淘汰时总是从 minFrequency 对应桶的第一个元素开始。
VISUAL FLOW
PRINCIPLE
STEP 1
新元素写入
put 时直接放入频次 1，对应代码会把 key 放进 `keysByFrequency[1]`，并把 `minFrequency` 重置为 1。
STEP 2
访问升级
onAccess 时先从旧频次桶移除，再写入新频次桶，访问次数 +1，形成“频次迁移”。
STEP 3
最小频次维护
旧桶被清空时，需要同步维护 `minFrequency`，否则淘汰时找不到正确的低频桶。
STEP 4
淘汰 victim
evictKey 只需读取 `minFrequency` 桶中的第一个 key。桶内部使用 `LinkedHashSet` 保留同频次的先后次序。
