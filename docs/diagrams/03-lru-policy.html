数据结构映射
LRU 在本项目中由抽象基类维护双向链表顺序。
nodeIndex 负责 key → Node 的 O(1) 定位；head / tail 是哨兵节点； 每次访问命中后，把节点移动到尾部，表示“最近刚使用”。当容量满时，直接淘汰 head 后面的节点。

1. put 时通过 append 把新节点放到尾部。
2. get 命中时调用 onAccess。
3. LRU 的 onAccess 会调用 moveToTail。
4. evictKey 永远弹出 head.next，也就是最久未使用的数据。

一句话总结
LRU 的本质是“顺序由最近访问时间决定”。本项目实现中， 访问命中后只做一件事：把节点移动到尾部；容量不足时只做一件事：删掉头部后继节点。 因为链表顺序和 HashMap 索引同时维护，所以访问、更新、淘汰都能保持在 O(1) 级别。
